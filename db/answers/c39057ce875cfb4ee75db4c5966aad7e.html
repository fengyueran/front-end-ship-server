优点:

  • 用来实现长连接，在 websocket 不可用的时候作为一种替代。
  • 跨域通信。
  • 无刷新上传文件，在 FormData 不可用时作为替代方案。
  • 创建一个全新的独立的宿主环境。

缺点:

  • 搜索引擎的爬虫还不能很好的处理 iframe 的内容，不利于 SEO。
  • iframe 会阻塞页面的加载，onload 事件不能及时触发(动态的给 iframe 添加 src 值可 解决) 。
  • iframe 和主页面共享连接池，而浏览器对同域的连接有限制，所以会影响页面的并行 加载(动态的给 iframe 添加 src 值可解决)。
  • 样式和脚本需要额外链入，增加服务器的 http 请求。
  • 多数小型的移动设备无法完全显示框架。
  • 浏览器的后退按钮无效。
